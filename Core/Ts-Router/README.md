# Vue3 + Vue Router 4 + TS 练习题

1.基于 TS 定义路由元信息类型（包含 title、requiresAuth 鉴权标识），配置 5 个路由（首页、登录、商品列表、商品详情、404），要求所有路由的 path/name/
meta 均通过 TS 类型约束。

2.实现全局路由守卫：判断 meta.requiresAuth，未登录时跳转到登录页，要求 TS 约束 to/from 参数类型，且守卫函数返回值符合 TS 类型要求。

3.实现路由懒加载：将商品列表、商品详情路由拆分为单独的代码块，要求 TS 约束懒加载组件的类型，且路由跳转时无类型报错。

4.实现动态路由：登录成功后，根据用户角色（管理员 / 普通用户）动态添加不同的路由，要求 TS 约束动态路由的类型，且添加后能正确渲染侧边栏。

5.封装一个通用的路由跳转函数（基于 useRouter），要求 TS 约束跳转的路由名称、参数类型，避免传错 path 或 params。
